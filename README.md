# Information Security Risk & Governance Case Studies

**IT 253 – Information Security | Southern New Hampshire University**  
**Author:** Francisco Sousa

## Overview

This repository contains three connected academic case studies demonstrating information security risk assessment, security planning, governance, and policy analysis.

The projects use a fictional enterprise and e-commerce environment to identify security weaknesses, evaluate control gaps, recommend practical improvements, and document security governance expectations.

Together, the artifacts demonstrate structured analysis relevant to information security, IT risk, governance, risk and compliance (GRC), and audit-oriented roles.

## Business Problem

Organizations must protect information assets while balancing business operations, regulatory expectations, and cybersecurity threats. These case studies examine how an organization can:

- Identify information security risks and control gaps
- Evaluate technical, administrative, and physical safeguards
- Improve accountability for access and system activity
- Establish a repeatable security-management process
- Strengthen resilience, recovery, and internet-facing protections
- Improve policy clarity and governance

## Case Studies

### 1. Information Security Risk Assessment & Action Plan

Assesses critical weaknesses within a fictional environment and recommends technical, administrative, and physical controls.

**Focus areas:**

- Threat and risk identification
- Centralized logging and monitoring
- Encryption, endpoint protection, and access accountability
- Security awareness training
- Business continuity and disaster recovery
- Business impact and control prioritization

[Open Project One – Risk Assessment (PDF)](documents/IT253_Project_One_Risk_Assessment.pdf)

### 2. Information Security Plan

Translates the assessment into a formal security plan with ownership, operating requirements, and control expectations.

**Focus areas:**

- Roles and responsibilities
- Security awareness and phishing exercises
- Named accounts, least privilege, MFA, and access reviews
- Vulnerability scanning and remediation timeframes
- Backup, restoration testing, and recovery planning
- DMZ, firewall hardening, TLS, IDS, and web application firewall controls

[Open Project Two – Security Plan (PDF)](documents/IT253_Project_Two_Security_Plan.pdf)

### 3. Information Security Policy Review

Reviews an acceptable-use policy and identifies practical policy improvements.

**Focus areas:**

- Password-management expectations
- Multi-factor authentication
- Incident-reporting responsibilities
- Vendor and third-party access considerations
- Roles, responsibilities, and policy clarity

[Open Information Security Policy Review (PDF)](documents/IT253_Information_Security_Policy_Review.pdf)
